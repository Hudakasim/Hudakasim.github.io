Autonomous Fire Detection & Extinguishing UAV

Graduation Project
Sep 2025 — Jun 2026
Team: Huda Kasimoglu, Ayse Nur Becit
Istanbul Sabahattin Zaim University

Overview

This graduation project develops an autonomous UAV system for early fire detection and targeted intervention. The drone detects smoke using a YOLOv8 deep learning model, calculates the GPS coordinates of the fire source using vision-based geo-location, and navigates autonomously to deploy a fire-extinguishing payload. The system was validated both in simulation (ArduPilot SITL + Gazebo) and through real flight tests.

The fully assembled UAV with Pixhawk, Raspberry Pi, camera, GPS, and telemetry modules mounted.

The Problem

The most critical stage in fighting fire disasters is the early detection phase before flames spread uncontrollably. Traditional methods only yield results when flames become visible, but tracking smoke provides much earlier data for locating the fire center. This project aimed to minimize time loss and human-induced risks in fire disaster response by building a fully autonomous detection and intervention system.

System Architecture

The system connects a drone (Pixhawk flight controller + Raspberry Pi 4B) to a ground station laptop running QGroundControl and the detection algorithm. The Raspberry Pi streams camera video via UDP to the laptop where YOLOv8 inference runs, while MAVLink handles telemetry and flight commands.

Software Stack

The software went through significant changes during the project. The autopilot was migrated from PX4 to ArduPilot due to a critical RAM limitation on the Pixhawk 2.4.8 (PX4 v1.16 consumed 98% of RAM). ROS 2 was also removed in favor of a lighter PyMAVLink-based pipeline to reduce complexity.

  • ArduPilot (ArduCopter) as the flight firmware
  • PyMAVLink for autonomous mission commands in GUIDED mode
  • OpenCV + YOLOv8 for real-time smoke detection
  • UDP H.264 video streaming from Raspberry Pi to laptop
  • MAVLink Router for telemetry forwarding to the detection script

YOLO Model Training

Two models were trained and compared: YOLOv8n and YOLO11n, both on the D-Fire dataset (26,000+ bounding boxes) for 70 epochs using Google Colab. YOLOv8n was selected for the final system due to fewer false detections in the smoke class (precision: 0.75, mAP@0.5: 0.74).

Target Coordinate Calculation

When smoke is detected, the system calculates the GPS coordinates of the fire source using a vision-based geo-location method. The camera is fixed at a 45° downward pitch angle (FOV: 114.6° horizontal, 98.6° vertical). Angular offsets from the image center are computed, then the ground distance is estimated using the drone's altitude from the barometer. Target GPS coordinates are calculated using a spherical Earth model.

The coordinate calculation is triggered when the horizontal pixel error drops below 5% of the frame width, confirming that the target is centered. The result is sent to ArduPilot as a GUIDED mode waypoint.

Terminal output showing successful target lock and calculated GPS coordinates.

Simulation

The full autonomous mission was validated in ArduPilot SITL with a custom Gazebo forest world before any real flight. The simulation covered takeoff, yaw-based scanning, smoke detection, target centering, coordinate calculation, autonomous navigation, and landing — all running through the same detect.py script used in real flights.

PID Tuning

The default PID values were unsuitable for the drone due to the frame's rigidity issues and vibration during flight. PID tuning was performed using Mission Planner log analysis, comparing desired vs actual values for roll, pitch, and yaw axes. After three iterative flight tests, stable flight was achieved.

Flight Tests

Full mission tests were performed using smoke flares as target sources. The test procedure included GPS recalibration, 1-meter and 4-meter takeoff tests, and then the full detection mission at 4 meters.

Results

In the best flight test, the drone successfully detected smoke and centered the target precisely. GPS drift caused slight landing offset. Two additional flights demonstrated correct smoke detection and centering. The final test ended in a crash due to unexpected deviation during a waypoint flight, breaking the landing gear.

Challenges & Lessons

  • PX4 v1.16 consumed 98% RAM on Pixhawk 2.4.8, requiring migration to ArduPilot
  • Battery calibration was impossible under PX4 but worked immediately on ArduPilot
  • Running YOLO + optical flow simultaneously on Raspberry Pi caused frame drops — solved by UDP streaming to laptop
  • School Wi-Fi was too slow for reliable video streaming — personal hotspot fixed quality issues
  • PID tuning required multiple iterations due to non-rigid frame causing vibrations
  • GPS drift affected coordinate accuracy in real-world conditions
